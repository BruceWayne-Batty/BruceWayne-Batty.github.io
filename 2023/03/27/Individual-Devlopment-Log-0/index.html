GameDesign Development Blog
March 27, 2023

Development Blog 0

This is the devlopment blog for the first week. We are a group of 5 people, with 3 of us(including me) from IMT(Interactive media technology) and 2 of us from CS(computer science). We hold 3 meetings last week.

In the first meeting, we created a discord group and introduced our own game experience and interests. And each of us posted some ideas for the second meeting.

In the second meeting, we created a Github repositories for the project and created a file for brainstorming.
Link for the ranking file
We listed 16 different game types(from the games we played before), and each group member ranked it using scale from 1-5 to decided which type of the game we will make.

In the third meeting, we chose 3 winner from the types, which are “Top down action RPG”, “Risk of rain like game” and “Deck building game rougelike”. we settled on a top down action 2d isometric game with a sort of apocalyptic theme. Jesper really wanted to work on some procedural generation so we might do that for levels. We also wrote down some more ideas in the brainstorm spreadsheet for the player character and weapons and such.

Laila came up with an idea involving the apocalyptic theme: The enemies are humanoid cyborgs with enhanced strength, speed and powers. Animal cyborgs are also an idea for example cyborg wolves that are more aggressive or birds that can fly. The setting could be a post-apocalyptic world (cities). The cyborgs could be part of an old army controlled by humans but that have turned against them. Below are some images generating using NovelAI base on the above description.



Inspired by this idea, I elaborated a little bit more on it and gave some general ideas of our character and enemy design: The characters can upgrade using parts of the cyborgs they defeat. And the idea of just picking some gears is way too normal, so I’m thinking if we can design the character to be some a universal model cyborgs. And when defeat the enemy, the player can just rip some parts of the enemy and directly use it as an upgrade. Following with this design, we can also make the player choose between differnet start universal model.

I’m reading a relative Call of Cthulhu module to get more inspiration for the theme. The name of the module is “Section M”.
