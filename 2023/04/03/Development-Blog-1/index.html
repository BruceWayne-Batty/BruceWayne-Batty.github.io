April 3, 2023

Development Blog 1

This is the devlopment blog for game “TechnoScourge: Rise of the Adaptivethe” second week. Two meetings and several online discussion were conducted.

For the first meeting, we focus on our game pitch. Vine drew the icon for the game and generated some other images of board setting using midjourney to choose which type of map we want to work on.

In the second meeting, we set up the milestone for each stage of the development, I choose to take part of the stat of the game.

For the discussion, we have some detail ideas about the story and mechanic of the game. Since the player will have a lot of freedom to make their own cyborgs, we want to make the player’s cyborg as the final boss for the next round(We plan to make the final boss has two stages, and the stage one will be the player in the last round).

I wrote a background story for the game, and this is the link to the story

When using the unity projects shared by Jesper, we encountered a problem of missing some prefab file, and the problem is solved by intalling blender and set blender as the default open method for the .blend file. After solving the problem, we used that project as the base to develope our game

I created an object type to store all kinds of data, now it only includes character data. The stats of enemy and inventory will be added later using the same type of object. Another script is created to fetch the data. I’ve tested fetching the speed in the player movement script and it works well. I committed the change to branch”Bruce”. I “accidentally” learned how to make the function of saving and loading data in the process of learning how to deal with data in the unity, so I will implement that later.

David said that We have an issue with the way the repository is structed right now. The .gitignore should ignore /Library/, but since the test project was placed in a subfolder /orthographics_test/ it was commited anyway. So we should fix that problem in our next meeting.
